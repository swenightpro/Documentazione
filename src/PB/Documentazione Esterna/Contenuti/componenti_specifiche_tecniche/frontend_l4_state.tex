\subsubsection{Layer 2 - State Management Layer (React Context + Custom Hooks)}

\paragraph{SessionContext}
\textbf{Descrizione del componente:} gestisce il lifecycle della sessione utente, lo stato real-time della conversazione tramite SSE, e il flusso di reset sessione con modale di conferma.

\textbf{Descrizione degli attributi:}
\begin{itemize}
    \item \texttt{sessionId}: identificativo della sessione corrente.
    \item \texttt{chatMessages}: storico messaggi della sessione attiva (espanso tramite SSE).
    \item \texttt{showNewSessionModal}: flag che definisce lo stato del pannello di reset modale.
\end{itemize}

\textbf{Descrizione dei metodi:}
\begin{itemize}
    \item \texttt{initSession()}: stabilisce o recupera la sessione attiva tramite sessionService.
    \item \texttt{confirmNewSession()}: svuota il carrello corrente, crea una nuova sessione e resetta i messaggi.
    \item \texttt{cancelNewSession()}: annulla l'apertura della finestra modale di nuova sessione.
    \item \texttt{handleNewSession()}: attiva la visualizzazione della finestra modale di conferma reset.
    \item \texttt{setChatMessages(...)}: setter per l'aggiornamento diretto dell'elenco messaggi.
\end{itemize}

\paragraph{CartContext}
\textbf{Descrizione del componente:} mantiene lo stato del carrello di sessione con aggiornamenti ottimistici e debouncing\ped{g} delle quantit\`a.

\textbf{Descrizione degli attributi:}
\begin{itemize}
    \item \texttt{cart}: righe prodotto attualmente presenti nel carrello.
    \item \texttt{pendingUpdates}: mappa debounce per accorpare aggiornamenti rapidi di quantit\`a.
\end{itemize}

\textbf{Descrizione dei metodi:}
\begin{itemize}
    \item \texttt{refreshCart()}: data-fetching\ped{g} forzato del carrello dal backend.
    \item \texttt{addItem(codArt, qta, extra)}: inserisce un articolo nel carrello tramite cartService.
    \item \texttt{removeItem(id)}: rimozione ottimistica dell'articolo con fallback\ped{g} server.
    \item \texttt{updateQty(id, qta, source)}: aggiornamento ottimistico con debouncing a 600ms.
    \item \texttt{clearCart()}: svuota il carrello nella UI locale.
    \item \texttt{hasPendingUpdates()}: verifica se esistono aggiornamenti di quantit\`a in attesa.
\end{itemize}

\paragraph{CustomHooks}
\textbf{Descrizione del componente:} fornisce API di accesso uniforme ai contesti e agli stati applicativi, riducendo duplicazione logica nei componenti UI.

\textbf{Descrizione degli attributi:} non sono esplicitati attributi nel diagramma.

\textbf{Descrizione dei metodi:}
\begin{itemize}
    \item \texttt{useSession() / useCart()}: espongono lo stato centrale e le firme interattive.
    \item \texttt{useSSE()}: espone le periferiche di ascolto event-stream reattivo push dal web server.
    \item \texttt{useProductSearch()}: wrappa la logica di debounce input per la ricerca prodotti nel catalogo.
    \item \texttt{useHealthCheck()}: gestisce periodic ping per rintracciare fallback\ped{g} di load downtime.
\end{itemize}

\paragraph{RoleBasedRendering}
\textbf{Descrizione del componente:} applica regole di rendering condizionale in base al ruolo, assicurando che le viste operatore non siano esposte ai clienti.

\textbf{Descrizione degli attributi:} nessun attributo di stato esplicitato nel diagramma.

\textbf{Descrizione dei metodi:}
\begin{itemize}
    \item \texttt{canRender(role: UserRole, feature: string)}: valuta la visibilita di una feature per un ruolo.
    \item \texttt{renderForRole(role: UserRole)}: compone la porzione UI consentita dal ruolo corrente.
\end{itemize}
